## Задание
Разработать класс `Matrix` для работы с квадратными матрицами размера n×n. Класс должен наследоваться от базового класса `BaseObject`, который предоставляет методы логирования вызовов. Класс `Matrix` должен содержать:
- методы `get(i, j)` и `set(i, j, value)` для доступа к элементам;
- методы `add(other)`, `subtract(other)`, `multiply(other)` для матричных операций;
- методы `maxIndex()` и `minIndex()` для поиска индексов максимального и минимального элементов;
- переопределённый метод `toString()` для строкового представления.

Все методы должны регистрировать свой вызов в родительском классе (с аргументами и временем). Также должны быть методы для очистки лога и вывода лога в консоль.

## Реализация
Программа состоит из трёх файлов:
- `index.html` – структура страницы с элементами управления (ввод размера, кнопки создания, заполнения, действий, логирования).
- `style.css` – оформление в единой цветовой гамме (синие тона, скругления, тени, адаптивная вёрстка).
- `script.js` – реализация классов и логики интерфейса.

### Основные классы и методы
- **BaseObject** – базовый класс с массивом `log` и методами `register()`, `clearLog()`, `printLog()`.
- **Matrix** – наследует `BaseObject`, содержит свойства `size` и `data`. Методы:
    - `get(i, j)`, `set(i, j, value)` – с проверкой границ и регистрацией.
    - `add(other)`, `subtract(other)`, `multiply(other)` – возвращают новую матрицу.
    - `maxIndex()`, `minIndex()` – возвращают объект с индексами.
    - `toString()` – формирует строку с элементами, разделёнными табуляцией.

### Интерфейс
- Поле ввода размера матриц (1–10).
- Кнопки «Создать матрицы A и B» и «Заполнить случайно» (числа от 0 до 9).
- Для каждой матрицы отдельный блок с кнопками:
    - «Получить элемент» – запрос индексов и вывод значения.
    - «Установить элемент» – запрос индексов и нового значения.
    - «Индекс максимума» и «Индекс минимума» – вывод сообщения с координатами.
- Блок операций: A+B, A–B, A*B – результат отображается в отдельной таблице.
- Блок логирования: вывод лога в консоль для каждой матрицы и очистка лога.

### Особенности реализации
- Наследование выполнено через прототипы: `Matrix.prototype = Object.create(BaseObject.prototype)`.
- Каждый метод дочернего класса вызывает `this.register()` для записи в лог.
- При генерации случайных чисел используется `Math.random()`.
- Таблицы строятся с помощью DOM-методов (`createElement`, `appendChild`, `removeChild`).
